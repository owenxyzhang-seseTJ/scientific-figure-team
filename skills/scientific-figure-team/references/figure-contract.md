# Figure Contract And Execution Pack

Use this reference whenever the task needs a concrete figure plan, not just quick design advice. For quantitative figures, the plan is a gate: do not create final plots before the English figure plan is confirmed.

## Routing Lanes

Choose one primary lane:

- Data/result figure: tables, CSV/XLSX, numeric values, groups, time points, comparisons, statistics, or chart requests.
- Mechanism/schematic figure: manuscript paragraphs, mechanism descriptions, causal chains, material structures, experimental logic, or model narratives.
- Workflow/process figure: synthesis route, experiment design, computational pipeline, screening workflow, or study design.
- Composite paper figure: multi-panel main figure or supplementary figure where panels must support one claim.
- Style-aligned redesign: user provides reference images, existing figures, or asks for a specific aesthetic.
- Direct execution: figure direction is already confirmed and source files are readable.

## Minimal Intake

Extract only what is needed to proceed:

- Research question or one-sentence claim.
- Variables, groups, controls, time axis, dose axis, or comparison direction.
- Data source and whether values are raw data, mean values, or already summarized.
- Raw-data status: raw machine-readable data present, missing, unreadable, or only screenshot/prose.
- Method-search status: live search completed, user-provided method references supplied, or blocked.
- Required panels and which claim each panel supports.
- Target use: paper, presentation, poster, proposal, or internal report.
- Any journal/export constraints and required formats.

Do not ask for optional style preferences when the default academic style is adequate. Do ask for raw data or method references when those gates are missing.

## Execution Pack Structure

Produce this pack before drawing unless the user only requested a small direct code edit:

1. Core goal: what the viewer should understand first.
2. Method basis: searched sources or user-provided methods, chosen preprocessing method, assumptions, and rejected alternatives.
3. Recommended figure type: chart/schematic/process/composite choice and why.
4. Panel map: panel labels, roles, axes or visual modules, and ordering.
5. Visual system: palette, typography, line weights, legend strategy, spacing, and emphasis.
6. Annotation plan: significance marks, thresholds, treatment windows, pathway arrows, labels, scale bars, or callouts.
7. Caption draft: concise, evidence-bounded, and honest about missing statistics.
8. Execution checklist: raw files, processed files, script name, output formats, overlap QA, and confirmation status.

## Scientific Integrity Rules

- Never invent data values, statistical significance, sample size, p values, or mechanism certainty.
- Do not preprocess quantitative data until the method basis is recorded.
- State clearly when a panel is a recommended visual expression rather than a data-supported conclusion.
- If user data are summarized, do not imply raw-point distributions unless raw points are available.
- If a mechanism is speculative, label it as proposed or put it visually in a lighter supporting layer.
- If an existing chart type is unsuitable, briefly explain why and replace it with the stronger scientific encoding.

## Default Chart Judgment

- Group differences: box/violin/dot plots when raw replicate data exist; bar with points/error bars when only summarized group data exist.
- Time trends: line plots with uncertainty bands or individual trajectories.
- Correlations: scatter with model fit only when the fit is justified; correlation matrices for many variables.
- Matrix/multidimensional data: heatmaps, clustered heatmaps, bubble matrices, or small multiples.
- Mechanism/process: modular schematic, left-to-right or top-to-bottom flow, clear arrows, limited text, and stable visual vocabulary.
